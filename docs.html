DOCUMENTAÇÃO

Guia completo para utilizar a API do Cyberpunk 2077. Todos os endpoints retornam dados em formato JSON.

Base URL

https://jose-pires-neto.github.io/Cyberpunk-2077-API/api/v1/
GET /characters.json

Retorna a lista completa de personagens do jogo.

Exemplo de Resposta

[
  {
    "id": 1,
    "name": "Vincent",
    "gender": "Male",
    "directory": "vincent",
    "images": [
      "https://...images/characters/sex/male/vincent/v01.png",
      "https://...images/characters/sex/male/vincent/v02.png"
    ],
    "description": "V, um pseudônimo para Vincent/Valerie...",
    "affiliation": "Mercenário"
  }
]

Campos

Campo Tipo Descrição
id number Identificador único
name string Nome do personagem
gender string Gênero (Male/Female)
images array URLs das imagens
description string Biografia do personagem
affiliation string Gangue/Corporação/Profissão
GET /gangs.json

Retorna informações sobre as gangues de Night City.

Exemplo de Resposta

[
  {
    "id": 1,
    "name": "Maelstrom",
    "images": ["https://...images/gangs/maelstrom/logo.png"],
    "description": "Gangue obcecada por cyberware..."
  }
]
GET /districts.json

Retorna os distritos de Night City.

Exemplo de Resposta

[
  {
    "id": 1,
    "name": "Watson",
    "images": ["https://...images/districts/watson/view.png"],
    "description": "Distrito decadente no norte..."
  }
]
GET /affiliations.json

Lista de afiliações disponíveis (gangues, corporações, profissões).

Exemplo de Resposta

[
  {
    "id": "maelstrom",
    "name": "Maelstrom",
    "category": "gang",
    "color": "#ff0000"
  },
  {
    "id": "arasaka",
    "name": "Arasaka",
    "category": "corporation",
    "color": "#ff0033"
  }
]

Exemplo de Uso

JavaScript (Fetch)

fetch('https://jose-pires-neto.github.io/Cyberpunk-2077-API/api/v1/characters.json')
  .then(response => response.json())
  .then(data => console.log(data));

Python

import requests

url = "https://jose-pires-neto.github.io/Cyberpunk-2077-API/api/v1/characters.json"
response = requests.get(url)
data = response.json()
print(data)

⚡ Rate Limits

Esta API é servida via GitHub Pages e não possui rate limits específicos. No entanto, recomendamos fazer cache local dos dados para melhor performance.
